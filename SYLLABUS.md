# C39 · Crunch Project Management — Syllabus

**Format:** 12 weeks · self-paced · open-source (GPL-3.0). Each week = 3 lectures, 3 exercises, 2 challenges, a mini-project, homework, and a quiz.

**Prerequisites:** You can use a web browser and a terminal. No prior project-management experience assumed — this course goes from zero to leading a delivery. Light SQL + Python are introduced where you need them for metrics and budgeting; C33 Crunch SQL is a helpful (not required) companion.

**Assessment (honor-based):** tick each lesson complete in the reader; finish the weekly mini-project and quiz. A certificate is issued on 100% completion to One-Stop members.

| Week | Theme | Mini-project |
|------|-------|--------------|
| 1 | PM foundations + the delivery lifecycle (scope/goals/success) | Write a project charter + one-page delivery plan |
| 2 | Agile, Scrum & Kanban — roles, ceremonies, when to use which | Stand up a Scrum + a Kanban board and compare |
| 3 | Backlog, user stories & requirements (INVEST, slicing, AC) | Turn a feature brief into a sliced, ready backlog |
| 4 | Estimation & sprint planning (points, capacity, commitment) | Plan a 2-sprint delivery to team capacity |
| 5 | Roadmapping & release planning (now/next/later, milestones) | Build a quarterly roadmap + release plan |
| 6 | Risk & dependency management (register, RAID, critical path) | Build a risk register + cross-team dependency map |
| 7 | Stakeholder & communication management (mapping, status, change) | Ship a stakeholder map + comms + status pack |
| 8 | Delivery metrics & flow — velocity, cycle time, burndown (SQL+pandas) | Compute a flow dashboard from real Jira/GitHub exports |
| 9 | Budgeting, resourcing & capacity — modeled in SQL, not Excel | Model project cost + burn + resourcing in PostgreSQL |
| 10 | Quality & release management (DoD, release, rollback) | Run a release with a DoD gate + rollback plan |
| 11 | Tooling + AI-assisted PM (GitHub Projects, Jira, AI workflows) | Automate a board + AI-draft a backlog and status |
| 12 | Capstone — lead a technology project end-to-end | Blank board to shipped, measured delivery |

**Outcome:** plan, run, and measure a technology project end-to-end — Agile, tooled, data-driven, and humane — and lead it in GitHub Projects and Jira.
