# Week 3 — Resources

Curated, free/official reading and tools for this week. Nothing here is required to have paid access — every link below has a free tier or is entirely free.

## Core reading (referenced in the lectures)

- **Ron Jeffries — "Essential XP: Card, Conversation, Confirmation"** — the original Three Cs formulation. <https://ronjeffries.com/xprog/articles/expcardconversationconfirmation/>
- **Mike Cohn / Mountain Goat Software — "User Stories"** — the role–goal–benefit template, straight from the source. <https://www.mountaingoatsoftware.com/agile/user-stories>
- **Bill Wake — "INVEST in Good Stories, and SMART Tasks"** — the original INVEST post. <https://xp123.com/articles/invest-in-good-stories-and-smart-tasks/>
- **Crisp (Henrik Kniberg et al.) — Story-splitting guides and cheat sheets** — patterns for vertical slicing. <https://www.crisp.se/gratis-material-och-guider/story-splitting>
- **Mike Cohn — "Advantages of Splitting User Stories Vertically"** — why horizontal slicing fails. <https://www.mountaingoatsoftware.com/blog/advantages-of-splitting-user-stories-vertically>
- **Cucumber (official) — "Gherkin Reference"** — the full Given/When/Then syntax reference. <https://cucumber.io/docs/gherkin/reference/>
- **Scrum.org — "Definition of Ready vs. Definition of Done"** — a clear breakdown of the two checklists. <https://www.scrum.org/resources/blog/definition-ready-vs-definition-done>
- **Atlassian — "MoSCoW Prioritization"** — official method overview with examples. <https://www.atlassian.com/agile/project-management/moscow-method>
- **Scaled Agile Framework (SAFe) — "Weighted Shortest Job First (WSJF)"** — the official WSJF reference, including the Cost of Delay breakdown. <https://scaledagileframework.com/wsjf/>
- **Atlassian — "Epics, Stories, Themes, and Initiatives"** — the backlog hierarchy vocabulary used in Lecture 2. <https://www.atlassian.com/agile/project-management/epics-stories-themes>

## Further, optional reading

- **Jeff Patton — "User Story Mapping" (overview article)** — the story-mapping technique referenced in Lecture 1 §4 and the mini-project's stretch goal. <https://jpattonassociates.com/story-mapping/>
- **Martin Fowler — "GivenWhenThen"** — a deeper dive on BDD-style acceptance criteria and common pitfalls. <https://martinfowler.com/bliki/GivenWhenThen.html>
- **Roman Pichler — "10 Tips for Writing Good User Stories"** — a practical checklist that overlaps with and extends INVEST. <https://www.romanpichler.com/blog/10-tips-writing-good-user-stories/>
- **Agile Alliance — "INVEST" glossary entry** — a quick community-maintained reference. <https://www.agilealliance.org/glossary/invest/>

## Tools to install / set up this week

You need **one** of the two below for the mini-project's live board requirement. Both are free for individual/small-team use.

- **GitHub Projects** (recommended if you already have a GitHub account from Week 1/2) — no install needed, works in-browser. Docs: <https://docs.github.com/en/issues/planning-and-tracking-with-projects>
- **Jira (Atlassian, free tier)** — up to 10 users free. Sign up: <https://www.atlassian.com/software/jira/free> · Getting started guide: <https://www.atlassian.com/software/jira/guides/getting-started/basics>

Optional, not required this week:

- **Miro (free tier)** — useful for the story-mapping stretch goal in Exercise 2 and the mini-project. <https://miro.com/>
- **draw.io / diagrams.net** — free, no-signup alternative for sketching a story map or workflow diagram. <https://app.diagrams.net/>

## A note on AI-assisted story writing

Later in this course (Week 11 — Tooling + AI-assisted PM) you'll cover using AI tools to draft backlog items directly. For this week, **write everything by hand** — the goal is building the judgment to catch a bad story, a horizontal slice, or a hollow benefit yourself, not to produce polished-looking output you can't yet evaluate. That judgment is exactly what makes AI-assisted drafting useful later instead of dangerous.
