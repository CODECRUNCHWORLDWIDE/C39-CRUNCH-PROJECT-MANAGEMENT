# Week 10 — Challenges

Two challenges, ~3 hours total, harder and more open-ended than the exercises. Both use Atlas's real GA data from the lectures — this is where the DoD gaps, open defects, and checklist blockers you queried all week finally have to turn into an actual decision, and where a release that goes wrong turns into something you can learn from instead of something you just survive.

| # | File | What it asks | Time |
|--:|------|--------------|-----:|
| 1 | [challenge-01-gate-a-risky-release.md](./challenge-01-gate-a-risky-release.md) | Run the real Atlas go/no-go with two open defects still on the board | 1.5h |
| 2 | [challenge-02-postmortem-a-bad-launch.md](./challenge-02-postmortem-a-bad-launch.md) | Run a blameless post-release review on a Northlight launch that actually went wrong | 1.5h |

## How to work these

Neither challenge has a single correct answer graded against an answer key — both are judged the way real release and incident decisions are judged: was the reasoning sound, did it use the real data instead of a gut feeling, and would a skeptical peer find the conclusion defensible even if they'd have leaned differently. Each file has its own rubric.

Do Challenge 1 before Challenge 2 — Challenge 2 assumes you're comfortable with the blameless-review structure from Lecture 3, which Challenge 1's go/no-go reasoning also draws on.

When you finish both, move on to the [mini-project](../mini-project/README.md).
